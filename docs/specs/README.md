## 🛠️ 개발 가이드라인

### 1. 코딩 컨벤션
- **네이밍**: 변수/함수는 `camelCase`, 컴포넌트/타입은 `PascalCase`, 상수는 `UPPER_SNAKE_CASE`.
- **상태 관리**: 복잡한 로직은 `services/`로 분리하며, UI 상태는 커스텀 훅을 통해 관리.
- **타입 시스템**: `any` 사용을 금지하고 모든 인터페이스를 `src/types/`에서 엄격히 정의.

### 2. 보안 및 성능 (2026 기준)
- **보안**: API 키는 절대 평문으로 저장하지 않으며, Web Crypto API를 통한 암호화 필수.
- **성능**: 팝업 로딩 300ms 이내, API 응답 2초 이내 유지를 목표로 함.
- **최적화**: 미사용 컴포넌트 트리쉐이킹 및 런타임 성능 모니터링 수행.
