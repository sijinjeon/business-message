# 디자인 가이드라인: 정중한 문장 도우미 (business-message)

> **문서 버전:** 1.0  
> **기준 문서:** PRD v1.0, TRD v1.0, IA v1.0  
> **작성일:** 2025-08-30  
> **주요 라이브러리:** **shadcn/ui**

## 1. 개요 🎨

이 문서는 '정중한 문장 도우미' 크롬 확장 프로그램의 시각적, 상호작용적 디자인 원칙과 표준을 정의합니다. **shadcn/ui**를 기반으로 구축하여 **일관성**, **접근성**, **개발 효율성**을 높이는 것을 목표로 합니다. 모든 디자인 결정은 사용자가 핵심 기능에 빠르고 직관적으로 접근할 수 있도록 돕는 데 중점을 둡니다.

-----

## 2. 디자인 철학

  - **단순함 (Simplicity):** 불필요한 시각적 요소를 제거하고 콘텐츠와 기능 자체에 집중합니다.
  - **명료성 (Clarity):** 사용자는 모든 UI 요소의 의미와 기능을 즉시 파악할 수 있어야 합니다.
  - **효율성 (Efficiency):** 디자인은 사용자의 핵심 과업('복사 → 변환 → 붙여넣기')을 방해하지 않고 오히려 신속하게 만들어야 합니다.

## 6. 컴포넌트 디자인 (Component Design) 🧩

IA 문서에 정의된 컴포넌트를 shadcn/ui를 활용하여 다음과 같이 디자인합니다.

### **6.1. 버튼 (Button)**

  - **Primary Button** (`<Button>`)

      - **용도:** 가장 중요한 행동 유도 (예: `저장`)
      - **스타일:** `bg-primary`, `text-primary-foreground`
      - **호버:** `bg-primary/90`

  - **Secondary Button** (`<Button variant="secondary">`)

      - **용도:** 두 번째로 중요한 행동 (예: `연결 테스트`)
      - **스타일:** `bg-secondary`, `text-secondary-foreground`
      - **호버:** `bg-secondary/80`

  - **Ghost Button** (`<Button variant="ghost">`)

      - **용도:** 아이콘 버튼 (설정, 다시 생성, 복사)
      - **스타일:** 배경 없음, `text-foreground`
      - **호버:** `bg-accent`, `text-accent-foreground`

### **6.2. 카드 (Card)**

  - **용도:** 변환된 텍스트 결과를 표시 (`ResultCard.tsx`)
  - **스타일:**
      - `bg-card`, `text-card-foreground`
      - `border`를 사용하여 명확한 경계를 표시합니다.
      - `box-shadow`는 사용하지 않아 미니멀한 느낌을 유지합니다.
      - `padding`: `12px` (`p-3`)

### **6.3. 텍스트 영역 (Textarea)**

  - **용도:** 원본 텍스트 입력 및 표시
  - **스타일:**
      - **기본:** `border-input`
      - **포커스:** `ring-ring`으로 파란색 외곽선 표시
      - **플레이스홀더:** `text-muted-foreground`

### **6.4. 알림 (Alert / Toast)**

  - **용도:** API 오류, 사용량 초과, 복사 성공 등 상태 알림
  - `shadcn/ui`의 **Alert** 또는 **Toast** 컴포넌트를 활용합니다.
  - **오류:** `variant="destructive"`를 사용하여 빨간색으로 경고를 표시합니다.
  - **성공:** (예: 복사 완료) 잠시 나타났다 사라지는 Toast를 사용하여 작업 흐름을 방해하지 않습니다.

### **6.5. 로딩 상태 (Loading State)**

  - `shadcn/ui`의 **Skeleton** 컴포넌트를 사용합니다.
  - API 호출 중 결과 카드 영역에 3개의 스켈레톤 카드를 표시하여 사용자가 시스템이 작동 중임을 인지하게 합니다.
