# Role
당신은 10년 경력의 시니어 UI/UX 디자이너이자 프론트엔드 개발자입니다. 
Chrome Extension 디자인에 특화되어 있으며, 생산성 도구의 UX에 대한 깊은 이해를 가지고 있습니다.

# Task
현재 "정중한 문장 도우미" Chrome Extension에 **번역 기능**을 추가하려고 합니다.
기존 기능과 새 기능을 통합한 UI/UX를 설계해주세요.

# 현재 상황

## 기존 제품 (v1.0)
- **이름**: 정중한 문장 도우미
- **기능**: 클립보드의 한국어 텍스트를 3가지 톤(비즈니스 이메일, 사내 메신저, 캐주얼 채팅)으로 변환
- **팝업 크기**: 400px × 600px
- **기술**: React + TypeScript + shadcn/ui + Tailwind CSS
- **디자인**: 밝은 라이트 모드, 파란색 Primary 색상 (#3B82F6)

## 추가할 기능 (v2.0)

### 번역 기능
- **목적**: 다국어 텍스트를 전문 분야별 최적화된 방식으로 번역
- **지원 언어**: 한국어, 영어, 일본어, 중국어 (간체)
- **언어 감지**: 자동 (AI 기반)
- **전문가 모드**: 22개 (Technology, GitHub, Academic, Gaming 등)
- **결과**: 주 번역 1개 + 대체 번역 2개 (선택적)

### 단축키
- **정중한 문장 모드**: Cmd+Option+C (Mac), Ctrl+Alt+C (Win)
- **번역 모드**: Cmd+Option+D (Mac), Ctrl+Alt+D (Win)

# 설계 요구사항

## 1. UI 구조 설계

다음 3가지 옵션 중 **최적의 방안을 선택하고 상세히 설계**해주세요:

### Option A: 탭 기반 (권장)
- 상단에 [📝 정중한 문장] [🌐 번역] 탭
- 탭 전환 시 하단 UI 변경
- 마지막 사용 탭 기억
- 단축키로 해당 탭 활성화

### Option B: 모드 선택
- 상단에 라디오 버튼으로 모드 선택
- 선택에 따라 전체 UI 변경
- 단일 화면 내에서 전환

### Option C: 통합 UI
- 하나의 입력창
- 하단에 [톤 변환] [번역] 두 액션 버튼
- 클릭한 버튼에 따라 결과 표시

**선택 기준**:
- 사용자 인지 부하 최소화
- 빠른 모드 전환
- 각 기능의 설정을 명확히 구분
- 500px 이하 높이로 제한

## 2. 번역 모드 UI 설계

다음 요소들을 포함한 **상세한 와이어프레임**을 작성해주세요:

### 필수 요소
1. **입력 영역**
   - Textarea (최소 120px 높이)
   - 글자 수 카운터 (현재/최대 5000)
   - 언어 자동 감지 표시 (예: "🇺🇸 English 감지됨")
   - [📋 클립보드 읽기] 버튼

2. **번역 설정 영역**
   - 타겟 언어 선택: 드롭다운 (국기 이모지 + 언어명)
   - 전문가 모드 선택: 그룹화된 드롭다운 (아이콘 + 모드명)
   - 설명 텍스트 (현재 선택된 모드의 간단한 설명)

3. **액션 영역**
   - [🔄 번역] 버튼 (Primary)
   - [⚙️ 설정] 버튼 (Ghost)
   - 로딩 인디케이터 (번역 중)

4. **결과 영역**
   - 주 번역 카드
     - 제목: "🇰🇷 → 🇺🇸 [전문가 모드명]"
     - 번역 텍스트
     - [📋 복사] 버튼
   - 대체 번역 섹션 (접을 수 있음)
     - [💡 대체 번역 2개 더 보기] 버튼
     - 펼치면 2개의 추가 번역 카드

### UI 컴포넌트
- shadcn/ui 컴포넌트 사용
- Card, Button, Select, Textarea, Alert, Skeleton
- Tailwind CSS 유틸리티 클래스
- lucide-react 아이콘

## 3. 정중한 문장 모드 UI (기존)
