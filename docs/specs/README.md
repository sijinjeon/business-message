## 🛠️ 개발 가이드라인

### 1. 코딩 컨벤션
- **네이밍**: 변수/함수는 `camelCase`, 컴포넌트/타입은 `PascalCase`, 상수는 `UPPER_SNAKE_CASE`를 사용합니다.
- **상태 관리**: 복잡한 비즈니스 로직은 `utils/` 또는 `services/` 레이어로 분리합니다.
- **타입 시스템**: `any` 사용을 지양하고 명확한 인터페이스를 정의합니다.

### 2. 보안 및 UX
- **보안**: API 키는 절대 평문으로 저장하거나 로그에 노출하지 않습니다. (AES-GCM 암호화 필수)
- **사용자 경험**: AI 호출 시 스켈레톤 UI 또는 로딩 상태를 제공하여 응답성을 보장합니다.
- **데이터 보호**: 사용자 입력 텍스트는 휘발성 메모리에서만 처리하며 외부로 유출하지 않습니다.

### 3. Git 및 협업
- **브랜치 전략**: `main` (배포), `develop` (통합 개발), `feature/` (기능 개발) 브랜치를 사용합니다.
- **커밋 메시지 (Conventional Commits)**:
  - `feat`: 새로운 기능 추가
  - `fix`: 버그 수정
  - `docs`: 문서 수정
  - `refactor`: 코드 리팩토링
  - `chore`: 설정 및 빌드 관련 수정
